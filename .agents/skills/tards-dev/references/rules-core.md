# Tards 核心规则

> 本文档只保留最高频引用的规则。完整规则见项目根目录 `Tards规则书1.0.docx` 和 `rules_text.txt`。

## 胜利条件

- 使对手 HP 降至 **0 或以下**。
- 双方 HP **同时** 降至 0 及以下 → **平局**。

## 棋盘

- **5 列**：高地(0)、山脊(1)、中路(2)、河岸(3)、水路(4)
- **5 行**：0-1 行为敌方区，2 行为禁行区（双方不可部署/操作），3-4 行为友方区
- **距离**：曼哈顿距离 `|r1-r2| + |c1-c2|`
- **前排**：同列中距离 row=2（中线）更近的异象算前排

## 回合流程

1. **抽牌阶段**
   - 第一回合：先手**跳过**抽牌，后手抽 1 张。
   - 其余回合：后手先抽 1 张，先手后抽 1 张。
   - 双方 T槽 +1（上限默认 10；离散 2 级沉浸度上限改为 8），然后**回满 T点**。
2. **出牌阶段**
   - 先手方在所有**奇数回合**中先行动。
   - 可执行：play（出牌）、bell（拍铃）、brake（拉闸）、exchange（兑换矿物）、exchange_squirrel（兑换松鼠）、set_attack_targets（预设攻击目标，含视野/高频/连击/多重打击）。
   - 拍铃：若本轮次未通过出牌改变过 T 点，拍铃时额外失去 1 T。
   - 拉闸：一方拉闸后，对方无法再拍铃。
3. **结算阶段**
   - 对战顺序：**水路(4) → 河岸(3) → 中路(2) → 山脊(1) → 高地(0)**
   - 同列内攻击者排序：**先攻等级降序 → 距离中线升序 → side（0 在前）**
   - 同先攻等级的异象**同时攻击**（通过 `EffectQueue.resolve()` 包裹）。
   - 异象攻击默认选中敌方前排；无前排则攻击对手英雄。

## 费用系统

| 符号 | 名称 | 性质 | 实现 |
|------|------|------|------|
| T | 通用点数 | 每回合自然获得，回合结束重置为 T槽值 | `t_point` / `t_point_max` |
| C | 兑换槽 | 不会自然增长；离散 2 级上限 4 | `c_point` / `c_point_max`，变动用 `c_point_change(delta)` |
| B | 鲜血 | 献祭友方异象获得，**回合结束清空** | `b_point` |
| S | 血契 | **跨回合保留，无上限** | `s_point` |

### 矿物（离散 1 级解锁）

- 铁锭(I)：兑换价 2CT，堆叠上限 4，打出获得 1T
- 金锭(G)：兑换价 3CT，堆叠上限 2，打出获得 2T
- 钻石(D)：兑换价 5CT，堆叠上限 1，打出获得 4T
- CT 规则：`XCT` 表示 X 点费用，每点可以是 1C 或 1T。支付时优先消耗 C，再消耗 T。

### 鲜血与献祭（2026-04-19 重构后）

- 部署带 B 费用的异象时必须献祭友方异象。
- 被献祭异象根据**丰饶等级**产血（默认丰饶 1）。
- 剩余鲜血保留到回合结束，之后清空。
- **献祭属于"消灭"**，会触发亡语；移除、返回手牌、洗入卡组均不触发。
- 冥刻异象默认具有**献祭 1、丰饶 1**。

**新流程：**
```
Game.action_phase()
  ├─ 筛选 valid_sacs（存活且 _sacrifice_remaining > 0）
  ├─ active.b_point += sum(丰饶)           [永久预加，不再回滚]
  ├─ active.sacrifice_chooser = lambda: valid_sacs
  ├─ card_can_play() → play_card() → cost.pay()  [统一扣除]
  └─ finally: 恢复 sacrifice_chooser

MinionCard.effect()
  └─ request_sacrifice(cost.b) → 消灭目标 → emit EVENT_SACRIFICE
       （effect() 不再操作 b_point）
```

## 流失生命值 vs 受到伤害

| 概念 | 代表表述 | API | 触发伤害替换 | 触发血契2级 |
|------|---------|-----|------------|-----------|
| **流失生命值** | "失去X点HP" | `Player.lose_hp(amount)` | ❌ | ❌ |
| **受到伤害** | "造成X点伤害" | `Player.health_change(-delta)` | ✅ | ✅ |

- 疲劳惩罚、血契1级沉浸度均使用 `lose_hp()`。
- 异象攻击英雄、策略造成伤害均使用 `health_change()`。

## 关键词精确定义

> ⚠️ **以下词条不可按字面理解**。日常语义 ≠ 游戏机制语义。

| 词条 | 定义 | 常见误解 |
|------|------|----------|
| **视野 X** | 出牌阶段需手动预设攻击目标列；可攻击本列及相邻 X 列的异象 | ❌ 不是"看穿一切"；❌ 不能看破潜水/潜行 |
| **潜水** | 结算阶段"无法被选中"（攻击自动落空）；**出牌阶段仍然可见可选中** | ❌ 不是全程隐身 |
| **潜行** | 攻击后获得"无法选中直到结算阶段开始"；与潜水结算阶段行为相同 | ❌ 同潜水 |
| **藤蔓** | 部署在友方异象上的 overlay；替宿主承受伤害；宿主死亡时同步死亡 | ❌ 不是独立目标 |
| **移动** | 内部效果回调（如将敌方异象移到友方区域）；**不是玩家行动** | ❌ 不是出牌操作 |
| **高频 X** | 结算阶段攻击 X 次，每次先攻 -1 | ❌ 不会自动要求选目标，除非同时有视野 |
| **协同** | 允许两个友方异象处于同一列的必要条件 | — |
| **独行** | 无法与友方异象处于同一列 | — |
| **坚韧 X** | 受到的伤害 -X（X 可以是负数 = 额外受伤） | — |
| **先攻 X** | 决定同列攻击顺序；高先攻先攻击；相同先攻同时攻击 | — |
| **亡语** | 异象被消灭时触发；**献祭属于消灭**；移除/返回手牌/洗入卡组不触发 | — |
| **回响 X** | 异象：生成 2T/1/1 复制体；策略：打出后返回手牌，回响等级 -1 | — |
| **绝缘** | 敌方策略无法直接以该异象为初始目标；AOE 策略需手动检查 `game.is_immune()` | — |
| **兴奋** | 若因攻击消灭了一个异象，则**再攻击一次** | — |

> **笔误澄清**："组队" = "协同"；"指令" = "策略"。
